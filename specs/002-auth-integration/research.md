# Research Findings: Better Auth + FastAPI JWT\n\n## Better Auth (Frontend/Next.js)\n- JWT plugin: Configurable secret (`BETTER_AUTH_SECRET`), HS256 default.\n- Claims: `sub` (user.id), `exp`, `iat`.\n- Storage: httpOnly cookies (secure).\n- Adapter: Postgres (Drizzle/Prisma-like).\nSources:\n- [Better Auth JWT](https://www.better-auth.com/docs/authentication/jwt)\n- [Next.js Setup](https://www.better-auth.com/docs/installation/nextjs)\n\n## Backend Verification (python-jose)\n- Lib: `python-jose[cryptography]` for HS256 decode.\n- Flow: `OAuth2PasswordBearer` → `jwt.decode(token, SECRET_KEY, algorithms=[&quot;HS256&quot;])` → extract `sub`.\n- Dep: `get_current_user` returns `user_id`.\nSources:\n- [FastAPI OAuth2 JWT](https://fastapi.tiangolo.com/tutorial/security/oauth2-jwt)\n- [FastAPI JWT Auth](https://testdriven.io/blog/fastapi-jwt-auth)\n- [python-jose Verify](https://medium.com/@kevinkoech265/jwt-authentication-in-fastapi-building-secure-apis-ce63f4164eb2)\n\n## User Model (SQLModel/Postgres)\n- Fields: `id`, `email`; hashing via Better Auth.\nSources:\n- [SQLModel Hashed Passwords](https://sqlmodel.tiangolo.com/tutorial/fastapi/update-extra-data)\n- [FastAPI JWT Postgres](https://neon.com/guides/fastapi-jwt)
