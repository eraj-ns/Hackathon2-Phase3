# Frontend Skill – Pages, Components & Styling

## Instructions

1. **Page Structure**
   - Semantic HTML layout
   - Header, main content, footer
   - Responsive grid or flex-based structure

2. **Component Design**
   - Reusable UI components
   - Clear separation of concerns
   - Props-based or modular structure

3. **Layout System**
   - Flexbox and Grid usage
   - Consistent spacing and alignment
   - Mobile-first responsiveness

4. **Styling**
   - Modern CSS (Flexbox, Grid)
   - Utility classes or component-level styles
   - Consistent color palette and typography

5. **Responsiveness**
   - Mobile, tablet, desktop support
   - Media queries or responsive utilities
   - Touch-friendly UI elements

## Best Practices
- Use semantic HTML elements
- Keep components small and reusable
- Follow mobile-first design
- Maintain consistent spacing and fonts
- Avoid inline styles where possible
- Ensure accessibility (labels, contrast, focus states)

## Example Structure
```html
<main class="container">
  <header class="header">
    <h1 class="page-title">Frontend Page</h1>
  </header>

  <section class="content-grid">
    <div class="card">
      <h2>Component Title</h2>
      <p>Reusable component content</p>
      <button class="primary-btn">Action</button>
    </div>
  </section>

  <footer class="footer">
    <p>© 2026 Frontend Skill</p>
  </footer>
</main>
